01 · Service Overview

서비스 개요

체체는 레뷰·강남맛집·미블 등 여러 플랫폼에 분산된 체험단 선정 내역을 한 곳에서 관리하는 2030 여성 크리에이터 전용 모바일 앱입니다.

Product Definition
당선된 체험단, 이제 앱 하나로
체험일자·가이드라인·마감일을 캘린더 한 장으로 보고, 협찬 품목을 금액으로 환산해 동기부여까지. 새로 뜬 체험단도 탐색 탭에서 바로 찾을 수 있습니다.
Problem
분산된 플랫폼, 놓치는 마감
레뷰·강남맛집·미블·서울오빠 등 플랫폼마다 선정 결과와 마감일이 달라 리뷰 업로드를 놓치거나 일정을 따로 수기로 관리해야 했다.
Solution
URL 한 번으로 자동 등록
선정 공고 URL 붙여넣기만 하면 체험명·마감일·가이드라인이 자동 완성. 캘린더에 즉시 반영되고 D-day 알림까지 자동 설정된다.
Vision
갓생 크리에이터의 필수 앱
이번 달 협찬 XX만원 절약, 리뷰 완료율 100% — 체험단 활동을 데이터로 시각화해 지속 활동 동기부여를 제공한다.
02 · Target & Persona

타겟 & 페르소나

체험단 활동을 통해 갓생을 살고 싶은 2030 여성이 코어 타겟입니다. 디자인은 토스 스타일의 클린하고 신뢰감 있는 방향으로 구현합니다.

👩‍💻
코어 페르소나
Primary Target
연령24~34세 여성
채널블로그·인스타 운영자
활동 빈도월 3~10개 체험단
사용 플랫폼레뷰·강남맛집·미블
Pain Points
!플랫폼마다 선정 알림이 달라 놓치는 경우 발생
!리뷰 마감일을 수기로 관리하다 기간 초과
!가이드라인을 메모장에 따로 저장해 관리 번거로움
!새 체험단 공고를 여러 사이트 돌아다니며 탐색해야 함
🎨
디자인 방향: 토스(Toss) 스타일 — 클린한 화이트 베이스, 명확한 타이포그래피, 컬러 포인트는 최소화하되 상태 뱃지에서 6색을 명확하게 사용. 모바일 퍼스트, 360~390px 기준. 브랜드 컬러는 네이비 #1C3C82를 메인으로, 빨강·파랑·노랑을 상태 컬러로 활용.
03 · Core Feature

코어 기능 — 체험단 일정 관리 캘린더

당선된 체험단의 모든 일정을 캘린더 한 장으로 보고 관리하는 기능. 이것이 체체의 존재 이유입니다.

CORE — P0
📅
월간 캘린더 뷰
선정된 체험단의 체험일자·리뷰 마감일이 캘린더에 자동 표시. 날짜 클릭 시 해당 일정 카드 목록 노출. 체험명이 셀 안에 직접 노출되어 한눈에 파악 가능.
CORE — P0
🗓️
주간 캘린더 뷰
이번 주 체험단 일정을 날짜별로 펼쳐서 확인. 체험일자·리뷰 마감일 2가지 이벤트 타입을 색상으로 구분 표시.
CORE — P0
🔔
D-day 기반 자동 알림
마감 D-3, 당일, 초과 시 자동 푸시 알림. 상태값도 날짜 기준으로 자동 전환 (선정됨 → 마감임박 D-3 → 기간초과). 서버 크롤링 없이 날짜 계산만으로 동작.
CORE — P0

체험 상세에서 "리뷰 완료" 버튼 1탭으로 완료 처리. 캘린더 셀에 완료 플래그 표시. 완료된 항목은 수익 집계에 자동 반영.

04 · Sub Features

서브 기능

캘린더 외 체체의 가치를 확장하는 기능들. 우선순위 순으로 정리합니다.

① 수익 환산 — 아낀 비용 시각화 (우선순위 2)
SUB — P1
💰
협찬 금액 직접 입력
체험 등록 시 협찬 품목 금액을 직접 입력. 월별 합산으로 "이번 달 XXX원 아꼈어요" 표시. 동기 부여 및 지속 활동 제고 효과.
SUB — P1
📊
수익 대시보드
협찬 상품가 + 광고비 분리 집계. 월별 완료 캠페인 수·평균 단가·총 절약 금액 표시. SNS 공유 카드 형태로 바이럴 유도 가능.
② 체험단 탐색 — 플랫폼 통합 검색 (우선순위 3)
SUB — P2
🔍
통합 공고 탐색
레뷰·강남맛집·미블 등 11개 플랫폼 공고를 한 화면에서. 리뷰매체·타입·카테고리·사이트 필터. 클릭 시 외부 플랫폼으로 이동.
SUB — P2
🗺️
지도 뷰
방문형 체험단을 Leaflet 지도에 마커로 표시. 플랫폼별 색상 구분. 카테고리 칩 필터. 마커 탭 시 하단 카드 표시.
SUB — P2
⚡
인원부족 필터
잔여 모집인원 2명 이하 체험단 즉시 탐색. 마감 임박 체험단을 빠르게 발견해 신청 기회 포착.
③ 체험단 등록 방식 — 3-Method (법적 안전 설계)
METHOD A — 즉시
🔗
URL 붙여넣기 자동 완성
선정 공고 URL 입력 → 공개 페이지 1회 fetch → 체험명·마감일·협찬품목·가이드라인 자동 완성. 약 10초 소요.
METHOD B — 단기
📤
공유하기 자동 추가
레뷰 앱에서 공유 버튼 탭 → "체체에 추가" 선택 → URL 자동 수신·파싱. iOS Share Extension / Android Intent Filter.
METHOD C — Phase 2
📷
AI 스크린샷 OCR
선정 알림 스크린샷 업로드 → Claude API로 필드 자동 추출. 지원 범위 밖 플랫폼을 위한 폴백 수단.
⚠️
법적 안전 원칙: 레뷰 이용약관 제6조·제9조 d·e 검토 결과, 플랫폼 마이페이지 크롤링(세션 쿠키 방식)은 전면 제거. 공개 캠페인 URL fetch만 허용. 상태 변경은 사용자 수동 입력 기반. (ADR-001 참고)
05 · User Flow

이용 플로우

신규 사용자가 앱 설치 후 첫 체험단을 등록하고 완료 처리까지 완료하는 핵심 여정입니다.

01
앱 설치 & 온보딩
스플래시 → 온보딩 3장(탐색·마감관리·수익) → 소셜 로그인(카카오·구글·애플) 또는 이메일 가입
온보딩 완료 후 캘린더 탭 기본 랜딩
체험단 미등록 시 "첫 체험단을 등록해보세요" 빈 화면 안내
02
체험단 당선 → 체체에 등록
레뷰 등 플랫폼에서 선정 알림 수신 후, 체체 앱으로 전환하여 캠페인 등록
방법 A (권장): 선정 공고 URL 복사 → 체체 앱 "+" → URL 붙여넣기 → 자동 완성 확인 → 저장
방법 B: 레뷰 앱 공유 버튼 → "체체에 추가" → 자동 등록
수기 입력: 직접 모든 필드 입력 (URL 없는 경우)
03
운영 정보 직접 입력 (필수)
URL 자동 완성 이후, 체험 운영에 필요한 정보를 직접 추가 입력
업주 전화번호 입력 (예약·방문 문의용)
체험일자 선택 (방문형의 경우)
예약 여부 선택 (F-08A): 예정 / 완료 / 불필요 — 기본값 "예정"
추가 메모 입력 (선택 — "주차 가능·사전 예약 필수" 등)
04
캘린더에서 일정 확인
등록 즉시 캘린더에 체험일자·리뷰 마감일 자동 표시. 월간/주간 뷰 전환 가능
캘린더 셀에 체험명 노출, 색상으로 이벤트 타입 구분
날짜 탭 → 상세 시트 직접 오픈 (이벤트 1개) 또는 선택 시트 경유 (복수)
상세 시트에서 가이드라인·전화번호·협찬금액 확인. 편집 버튼으로 전체 필드 수정 가능
05
D-day 알림 수신 → 리뷰 업로드
마감 D-3, 당일에 자동 푸시 알림 수신. 알림 탭 → 해당 캠페인으로 딥링크
알림 센터에서 D-3·D-1·당일 알림 확인 및 딥링크 이동
상태가 자동으로 "마감임박"으로 변경되어 시각적 긴박감 표시
06
리뷰 완료 처리 → 수익 반영
리뷰 업로드 후 체체 앱에서 "리뷰 완료" 체크 1탭
캘린더에 완료 플래그(🚩) 표시 → 상태 "리뷰완료"로 전환
협찬 금액이 이번 달 수익 대시보드에 자동 합산
마이페이지 "이번 달 절약 금액" 업데이트
06 · Calendar Field Definition

캘린더 상세 필드 정의

각 체험단 등록 시 저장되는 필드 전체 목록. 입력 방식에 따라 3가지 레이어로 구분합니다.

# 필드명 입력 방식 설명 필수/선택
F-01체험명자동 완성캠페인 공고 제목. URL 파싱 시 자동 추출. 수기 수정 가능.필수
F-02플랫폼자동 완성레뷰·강남맛집·미블 등. URL 도메인으로 자동 분류. 수기 선택 가능.필수
F-03리뷰 마감일자동 완성D-day 계산 기준. 캘린더에 빨간 점·알림 발송 기준. 수기 수정 가능.필수
F-04협찬 품목자동 완성제공 상품·서비스명. 공고 혜택 섹션에서 파싱. 수기 수정 가능.필수
F-05협찬 금액직접 입력협찬 품목의 시장 금액. 수익 대시보드 집계 기준. 원 단위 숫자 입력.필수
F-06체험 타입자동 완성블로그·인스타·클립 등 리뷰 채널 종류. 복수 선택 가능. 칩 UI.필수
F-07가이드라인자동 완성해시태그·필수 포함 문구·포스팅 조건 등. 공고 본문에서 파싱. 수기 수정 가능.선택
F-08 v1.1 주말 방문 가능 여부 자동 완성 공고 내 주말 가능 여부 명시 시 자동 파싱. 가능 / 불가 선택 UI (기본값 없음, 미선택 허용). 수기 입력 가능. 미입력 시 상세 화면에 —(공란) 표시. 선택
F-08A 신규 예약 여부 직접 입력 방문형 체험단의 매장 예약 진행 상태. 예정 / 완료 / 불필요 3단계 선택. 기본값 "예정". 완료(초록)·예정(파랑)·불필요(회색)로 색상 구분. 배송형은 "불필요" 선택. 선택
F-09업주 전화번호직접 입력매장 예약·방문 문의용. 탭 시 전화 앱 연결. 배송형은 해당 없음.필수 (방문형)
F-10체험일자직접 입력방문·체험 예정일. 캘린더에 초록 점으로 표시. 날짜 피커 UI.필수 (방문형)
F-11리뷰 완료 여부직접 입력체크박스/버튼 1탭. 완료 시 상태 "리뷰완료"로 전환, 수익 집계 반영.필수
F-12추가 메모선택 입력주차 가능 여부, 사전 예약 필요, 기타 체험 메모. 자유 텍스트 최대 200자. 상세 시트에서 항상 즉시 편집 가능.선택
💡
등록 UX 원칙: URL 파싱으로 F-01~F-08 자동 완성 후, 사용자는 F-08A(예약여부)·F-09·F-10을 추가 입력하면 등록 완료. F-08(주말방문)은 선택 항목으로 미입력 허용. 전체 등록 소요 시간 목표 30초 이내.
